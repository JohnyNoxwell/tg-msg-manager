# Roadmap Documentation

## Current roadmap

- [`ROADMAP.md`](ROADMAP.md) - current roadmap and post-foundation future directions.

## Next steps

Next steps must be expressed as active stage task files before implementation starts.

## Backlog

Backlog items are planning inputs. They are not active work by themselves.

Deprecated backlog files are archived under [`../archive/deprecated_stage_files/`](../archive/deprecated_stage_files/).

## Deferred

Deferred items may describe product ideas, architecture ideas, or historical priorities. They require explicit reactivation through an active task.

## Rules

- Roadmap items are not implementation permission.
- Implementation requires an active stage task under [`../stages/active/`](../stages/active/).
- Do not implement analytics, OSINT/profiling, OCR, speech-to-text, media analysis, schema changes, GUI/dashboard/SaaS, or other product features from roadmap text alone.
- If a roadmap item conflicts with `AGENTS.md` or current architecture docs, stop and report.
