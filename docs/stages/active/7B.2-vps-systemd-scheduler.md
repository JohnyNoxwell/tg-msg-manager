## 1. PURPOSE

Add production VPS systemd timers for scheduled `tg-msg-manager update` and destructive `clean --apply --yes` runs through a shared non-blocking scheduler lock.

## 4. ATOMIC IMPLEMENTATION TASKS

1. Add POSIX `sh` launcher `deploy/vps/run-scheduled.sh` with `set -eu`, argument forwarding to `/opt/tg-msg-manager/deploy/vps/tgm`, and one shared non-blocking `flock` lock at `/run/lock/tg-msg-manager-scheduler.lock`; if busy, write a clear stderr message and exit `75`.
2. Add systemd units under `deploy/vps/systemd/` for update and clean services/timers.
3. Configure update service to run `/opt/tg-msg-manager/deploy/vps/run-scheduled.sh update`.
4. Configure clean service to run `/opt/tg-msg-manager/deploy/vps/run-scheduled.sh clean --apply --yes`.
5. Configure timers with explicit `Europe/Kyiv` `OnCalendar`: hourly at `:00` for update and daily `04:30` for clean; set `SuccessExitStatus=75` on services.
6. Add `deploy/vps/install-systemd-schedules.sh` to install units into `/etc/systemd/system`, run `systemctl daemon-reload`, enable/start both timers, and print verification commands without manually starting clean.
7. Update `deploy/vps/README.md` with schedule, destructive `tgd` note, verification commands, and rollback commands.
