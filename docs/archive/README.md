# Archive

## Purpose

The archive preserves historical notes, old prompts, superseded plans, and deprecated stage files.

Do not use archived files as current instructions unless the user explicitly asks.

## What is archived here

- [`legacy_notes/`](legacy_notes/) - historical baselines, old indexes, closed local notes, and superseded smoke notes.
- [`old_prompts/`](old_prompts/) - brainstorms and prompts that are not current guidance.
- [`deprecated_stage_files/`](deprecated_stage_files/) - old backlog and stage planning files.

## How agents should treat archived files

- Do not read archive files by default.
- Do not execute archived prompts.
- Do not use archive content to justify product features.
- Treat archived references as historical and possibly stale.
- If the user explicitly asks for archived context, read only the relevant file.

## Legacy notes

Legacy notes may contain useful history, but current architecture and development guidance lives under:

- [`../architecture/`](../architecture/)
- [`../development/`](../development/)
- [`../stages/reports/`](../stages/reports/)

## Old prompts

Old prompts are preserved for history. They are not active tasks.

## Deprecated stage files

Deprecated stage files are not executable. Current executable tasks live only under [`../stages/active/`](../stages/active/).
